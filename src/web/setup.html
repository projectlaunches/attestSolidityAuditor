Anvil

Fresh local EVM; Foundry also powers authorized pinned-block forks.

Checking…

Anvil is installed with Foundry. The auditor can launch an opt-in disposable loopback chain, validate bounded constructor inputs against compiled ABI metadata, deploy an eligible target, verify its receipt and bytecode, snapshot it, and shut it down. Separately authorized fork checks use built-in public Ethereum, Base, or BNB profiles and never broadcast.

Project setup

Install and start this project

Project dependencies and verified Aderyn

Continue here after Node, Foundry, and Slither are available.

Project command 
npm ci --ignore-scripts
npm run deps:audit
npm run tools:install-aderyn
npm test
npm run launch

The tester launch opens the private session once in the default browser. If the workstation blocks automatic launch, open $HOME/.local/share/attest/open-attest.html. Developers can use npm start or npm run dev without opening extra windows. The app stays on loopback and never broadcasts externally; the explicit Anvil option may deploy only to a disposable local development chain.

Harness-aware

Optional and advanced engines

Host-installed tools are detected by the app but are not invoked blindly. The recommended Mythril and Echidna containers are intentionally marked for manual verification because the current probe checks host commands, not Docker images. Installation never means a property, solver, or witness is ready for a particular contract.

Mythril

Bounded multi-transaction symbolic analysis.

Container — manual check
docker pull mythril/myth
docker run --rm -v "$PWD":/tmp mythril/myth analyze /tmp/src/Target.sol

Docker isolates Python-version conflicts. Replace src/Target.sol with the contract path inside the mounted project. This container is not included in live host-command detection. Runs must receive transaction-depth and time bounds before product integration.

Official Mythril repository ↗

Echidna

Stateful property fuzzing and minimized counterexamples.

Container — manual check
docker pull ghcr.io/crytic/echidna/echidna
docker run --rm -it -v "$PWD":/src ghcr.io/crytic/echidna/echidna

Requires meaningful Solidity properties. The official container is intended for x86 systems and is not included in live host-command detection.

Product flow

How to run an evidence-backed audit

  1. Load source.Paste or select one .sol file. Imports are detected but never downloaded automatically.
  2. Declare intent.Choose the product type and describe trusted roles, intentional behaviors, and accepted risks.
  3. Choose AI scope.Sign in with ChatGPT and enable AI only when sending the source for review is acceptable.
  4. Review evidence.Run the audit, inspect completed and skipped engines, challenge classifications, then export Markdown or JSON.
Install only the tools needed for the applicable properties. A larger tool count is not automatically stronger evidence.
